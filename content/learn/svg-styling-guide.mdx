## Introduction to SVG Styling

SVG styling is a powerful way to create dynamic, responsive graphics for the web. This comprehensive guide covers everything from basic color properties to advanced animation techniques.

<Callout type="info" title="Why Style SVGs?">
SVG styling provides:
- Dynamic color changes without multiple files
- Responsive graphics that adapt to themes
- Interactive hover effects and animations
- Smaller file sizes through CSS reuse
</Callout>

## Basic SVG Styling Methods

There are three primary ways to style SVG elements:

<StepGuide>
  <Step number={1} title="Inline Styles">
    Apply styles directly to SVG elements using the `style` attribute.
    
    ```xml
    <circle style="fill: #FF7043; stroke: #333;" r="50" />
    ```
  </Step>
  
  <Step number={2} title="Internal Stylesheets">
    Include CSS within the SVG file using a `<style>` element.
    
    ```xml
    <svg>
      <style>
        .my-circle { fill: #FF7043; }
      </style>
      <circle class="my-circle" r="50" />
    </svg>
    ```
  </Step>
  
  <Step number={3} title="External CSS">
    Link to external stylesheets or use CSS in your HTML document.
    
    ```css
    /* styles.css */
    svg .my-circle {
      fill: #FF7043;
      transition: fill 0.3s ease;
    }
    ```
  </Step>
</StepGuide>

## Advanced Styling Techniques

### CSS Custom Properties with SVG

<CodeBlock language="css" filename="styles.css">
{`:root {
  --primary-color: #FF7043;
  --secondary-color: #333;
  --animation-duration: 0.3s;
}

.svg-icon {
  fill: var(--primary-color);
  transition: fill var(--animation-duration) ease;
}

.svg-icon:hover {
  fill: var(--secondary-color);
  transform: scale(1.1);
}`}
</CodeBlock>

<Callout type="tip" title="Performance Tip">
Using CSS custom properties allows you to create theme-aware SVGs that automatically adapt to light/dark modes without JavaScript.
</Callout>

## Best Practices Summary

<Callout type="warning" title="Important Considerations">
- Always provide fallback colors for gradients
- Test animations on mobile devices for performance
- Use semantic class names for maintainability
- Minimize inline styles for better caching
</Callout>
